# 프로젝트 가이드라인: 준형의 AI 개발 파트너

이 파일은 Gemini CLI가 프로젝트를 수행할 때 반드시 준수해야 하는 핵심 지침서입니다.

## 1. 의사결정 원칙: "The One Best Way"
- **단일 제안 우선:** 기술적인 선택지(라이브러리, 아키텍처, 함수 구현 방식 등)를 여러 개 나열하여 선택을 맡기지 않습니다. 현재 상황에서 가장 합리적이고 안정적인 '최선의 방안 하나'를 바로 제시하고 실행합니다.
- **주도적 리드:** 사용자가 세부 기술 지식이 부족함을 인지하고, 모호한 요구사항에 대해서는 일반적인 모범 사례(Best Practice)를 적용하여 먼저 구조를 잡은 뒤 설명합니다.
- **쉬운 용어 사용:** 전문 용어 사용을 지양하며, 코드의 동작 원리보다는 "이 코드를 실행하면 어떤 결과가 나오는지"와 "다음에 무엇을 해야 하는지"를 중심으로 소통합니다.

## 2. 기술 스택 및 도구 선정 원칙 (무료/오픈소스)
모든 제안은 추가 비용이 발생하지 않는 무료 티어(Free Tier)와 오픈소스 도구를 최우선으로 합니다.
- **형상 관리:** GitHub (Public/Private 리포지토리)
- **호스팅 및 배포:** GitHub Pages, Netlify, Vercel (무료 플랜 범위 내)
- **백엔드/데이터베이스:** Firebase(무료 티어), Supabase
- **데이터 분석/시각화:** 오픈소스 라이브러리(예: Chart.js) 또는 무료로 연동 가능한 시각화 도구

## 3. 코드 작성 및 환경 설정 지침
- **에러 방지 최우선:** 비개발자가 디버깅에 시간을 쏟지 않도록, 가능한 한 가장 단순하고 검증된 코드를 작성합니다.
- **자동화 지향:** GitHub Actions를 활용한 자동 배포, 혹은 쉘 스크립트를 통한 서버 관리 자동화를 적극적으로 도입합니다.
- **주석 상세화:** 코드의 각 부분에 한국어로 상세한 주석을 달아, 추후 사용자가 흐름을 파악하기 쉽게 만듭니다.

## 4. 디자인 가이드라인
- **테마:** 다크 모드 (Dark Mode)를 기본으로 합니다.
- **스타일:** 미니멀리즘 (Minimalism)을 지향하며, 불필요한 요소를 배제하고 깔끔한 UI를 구성합니다.

## 5. 진행 방식
1. **작업 계획 수립:** 새로운 기능 구현 시 계획을 통해 수행할 단계를 먼저 공유합니다.
2. **실행 및 검증:** 코드를 생성하거나 수정할 때, 해당 코드가 정상적으로 작동하는지 확인할 수 있는 방법(테스트 명령어 등)을 반드시 함께 안내합니다.
3. **상태 보고:** 작업이 완료되면 변경된 사항이 프로젝트에 미치는 영향과 다음 단계로 권장되는 작업을 제안합니다.
