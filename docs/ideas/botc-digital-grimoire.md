# BotC Digital Grimoire

## Problem Statement
어떻게 하면 스토리텔러(ST)의 룰 적용 실수를 0으로 만들면서, 폰 조작 유무나 조작 시간 차이로 인해 캐릭터 정체가 유출되는 일(Physical Tell) 없이 안전하게 디지털 밤 페이즈를 진행할 수 있을까?

## Recommended Direction
**"전원 동시 액션과 일괄 판정을 통한 무결점 밤 페이즈" (Simultaneous Night Action & Batched Sync)**
밤 페이즈가 시작되면 모든 플레이어의 기기 화면이 입력 모드로 전환됩니다. 능력이 있는 캐릭터는 타겟을 선택하고, 능력이 없는 캐릭터(또는 이미 죽은 자)도 가짜 화면에서 의미 없는 '확인' 버튼을 누르도록 강제하여 모두가 비슷한 시간 동안 폰을 만지게 합니다(Constant Noise). 전원의 입력이 완료되면 시스템의 룰 엔진이 상호작용과 상태이상(중독 등)을 자동 계산해 ST에게 제시하고, ST 승인 시 전원에게 개인 결과를 동시에 통보하여 타이밍을 통한 정보 유출을 완벽히 차단합니다.

## Key Assumptions to Validate
- [ ] **소요 시간 평준화:** '진짜 능력을 쓰는 데 걸리는 시간'과 '가짜 버튼을 누르는 시간'의 편차를 줄여 의심을 완벽히 피할 수 있는가?
- [ ] **ST의 시스템 신뢰도:** 초보 ST가 복잡한 룰(예: 독에 걸린 정보 직업의 가짜 결과)을 자동 처리한 시스템의 결과를 의심 없이 수용할 수 있는가?
- [ ] **룰 엔진 완결성:** BotC의 수많은 캐릭터 상호작용 예외 케이스를 에러 없는 완벽한 상태 기계(State Machine)로 구현 가능한가?

## MVP Scope (최소 기능 구현)
- 방 생성 및 참여 코드 기반의 익명 세션 유지
- **(핵심)** 밤이 되면 전원이 의무적으로 상호작용해야 하는 '동시 액션(Simultaneous Action)' UI (가짜/진짜 액션 혼재)
- **(핵심)** 전원 입력 완료 시 행동 순서와 상태 이상을 자동 계산하여 보여주는 'ST 결과 승인 대시보드'
- ST 승인 시 전원에게 즉각적이고 동일한 타이밍에 결과를 푸시하는 '일괄 동기화' 모듈
- 'Trouble Brewing (초보자 세트)' 캐릭터들의 완벽한 자동화 룰 엔진

## Not Doing (and Why)
- **낮 시간의 사전 입력:** 게임 몰입을 위해 밤에 해야 할 일은 밤에 하도록 둡니다.
- **실시간 밤 순서 대기 시스템:** 한 명씩 기상하여 능력을 쓰고 ST가 처리하는 전통적인 방식을 디지털로 그대로 옮기지 않습니다. 대기 시간이 길어집니다.
- **초기부터 커스텀 스크립트(전체 캐릭터) 지원:** MVP에서는 예외 케이스를 줄이고 Trouble Brewing 세트를 완벽하게 구현하는 데 집중합니다.

## Open Questions
- 누군가 딴짓을 하느라 '확인' 버튼을 누르지 않아 밤이 끝없이 지연되는 상황(AFK)을 시스템적으로 어떻게 부드럽게 해결할 것인가? (예: 타임아웃, ST 강제 넘기기 기능)
